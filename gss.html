Insights

from Users

• If we could provide a platform to the user where he can edit the code and view its after effects simultaneously then we can reduce the possible errors that occur like missing files, wrong location etc.
• If the user can see (visualize) the simulation geometry he can mention the various boundary conditions correctly without missing on to something.
• If the user can be provided with extra tools than he can explore his different ideas regarding simulation without any boundary limitation.
• If the system could remember the operations he mostly perform or the input parameters he mostly use or he last entered than his effort can be reduced as he mostly performs repeated tasks
• the cognitive load of user and human errors can be decreased by gamification of process

Persona


Ishan Sharma

Professor, Department of mechanical engineering, IIT Kanpur


User type: Thinker
Problem: His imagination is limited by lack of suitable platform
Observation: prefers to visualize practically
His vision: He wants to explore the possibilities He wants visualization software easy to use, assistive which could further be developed and extended to various models (currently the simulator uses only cuboid and trapezoidal geometry, but in future this could be extended to more complicated geometry like zigzag and inclined)

Prasad Sonar

PhD student, Department of mechanical engineering, IIT Kanpur


User type: Coder
Observation: User doesn’t explore and create something from scratch instead he works on input menu and command shortcuts
Problem: Unable to load previous session, repeated use of command prompt, missing files, No information, feedback and assistance from the system, sometimes he gets lost in the process itself as there is no gamification of process.
His vision: An assistive umbrella platform that allows him to go a step forward and backward at any point of time.
